## 3-2장

## 코딩 스타일

### 알게된 점

1. 개발자는 가능한 한 간결하고 읽기 쉽게 코드를 작성해야함
2. 복잡한 문제를 간결하고 사람이 읽기 쉬운 코드로 작성해 해결하는 것이 진정한 프로그래밍 기술
3. ! 무조건 따라야 할 규칙은 없음

문제 1

```javascript
if (condition) {
  //코드 1
  // 코드 2
}
```

- 여러줄로 코드가 짜여질때에는 중괄호를 쓰는 것을 추천
- 여러줄일때 중괄호를 붙이지 않는다면 오류가 난다.

문제 2

```javascript
if (condition) alert("무야호!");
```

- 한줄로 코드가 끝날경우 중괄호를 붙이지 않는 것을 추천

### 느낀점

1. 코드를 짤때 정말 다른 사람이 봐도 알아볼수 있을 정도로 간편하게 짜야한다는 것을 느낀다.
2. 꼭 지켜야 할 규칙은 없지만, 사람들이 자주 사용하는 것을 따라하는것도 나쁘지 않은 듯하다.
